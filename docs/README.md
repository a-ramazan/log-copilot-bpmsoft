# `docs`

## Назначение

Здесь лежит основная проектная документация: архитектура, контракты, workflow команды и краткие ТЗ по зонам ответственности.

## Что на входе

- согласованные инженерные решения
- правила по входу, выходу и ownership

## Что на выходе

- понятные инструкции, по которым можно сразу работать без лишних договорённостей

## Важно

- документация должна совпадать с кодом и тестами
- при изменении контракта нужно обновлять docs в том же PR
